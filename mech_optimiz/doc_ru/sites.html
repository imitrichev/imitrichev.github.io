Спецификация частиц, занимающих несколько активных центров

Cantera (и mech_optimiz) имеет поддержку поверхностных реакций, включающих частицы, занимающие несколько активных центров на поверхности (multiple site species). Для конвертера файлов формата cti в формат ctml (cti2ctml) данная возможность добавлена в декабре 2005 года Prof. D. Goodwin (<a href ="http://code.google.com/p/cantera/source/browse/cantera/trunk/interfaces/python/ctml_writer.py?r=1906#564" style="color: #FF9933>"строки кода, подтверждающие это)

if self._size != 1.0:
s.addChild("size",self._size)

Определение частиц, занимающих несколько активных центров, в cti-файле

Конвертер cti2ctml, переводящий файлы cti в ctml-формат, корректно обрабатывает опцию size (подробнее, см. <a href ="https://groups.yahoo.com/neo/groups/Cantera/conversations/topics/1084" style="color: #FF9933>" ответ Prof. D. Goodwin . Таким образом, в cti-файлах возможно использовать опцию size = <число> при объявлении частиц. Например, кислород O2(S1), занимающий 2 активных центра, объявляется следующим образом:

species(name = 'O2(S1)', atoms = 'surf_site:2', size = 2, ...)

Эта возможность используется в mech_optimiz, поскольку взаимодействие mech_optimiz и Cantera осуществляется, в том числе, путем создания объекта класса "идеальный газ" (IdealGas) Cantera, который, в свою очередь, использует xml(ctml)-файл с кинетическим механизмом.

Определение частиц, занимающих несколько активных центров, в inp-файле

В inp файле объявить частицу, занимающую два активных центра, можно так:

O2(S1)/2/

Данный формат записи полностью поддерживается программным пакетом Surface Chemkin. Для использования в mech_optimiz, в .inp-файл следует добавить также строки, характеризующие, сколько атомов активной поверхности включено в активный центр данного типа. Так как два активных центра имеют другое число атомов, их надо объявить отдельно, маркировав, например, как S1:

!ALIAS (S) Ce:1
!ALIAS (S1) Ce:2
!ALIAS (S2) Au:1

Ключевое слово !ALIAS позволяет не указывать атомы активного центра в названии частицы и не требует наличия обязательного объявления частиц в совокупности с коэффициентами термодинамических полиномов (например, в файле mech.dat) для определения атомов в частице.

Перевод единиц предэкспоненциальных множителей

Необходимость в переводе единиц возникает, когда в кинетической схеме присутствуют реакции порядка, выше первого. Когда в реакции участвуют 2 и более реагента, предэкспоненциальный множитель обычно приводят в размерности константы скорости, а не в \(с^{-1}\), как для реакций первого порядка. Чтобы получить значения предэкспоненциального множителя \(A_s\) в \(с^{-1}\) для реакции второго порядка, необходимо умножить A на число активных центров \(\Gamma\), [моль/м2], для данного катализатора:

\[ A_s = A_{dim} \Gamma \]

Если некоторый реагент занимает несколько активных центров, A, приведенное в размерности константы скорости, не изменяется (этот принцип используется и в Cantera, и в mech_optimiz). \(A_{dim} = const\), \(k_{dim}=const\).
Это приводит к тому, что для аналогичных кинетических параметров и степеней заполнения поверхности реагентами скорость реакции, включающей один из реагентов, занимающий два активных центра, на единицу поверхности уменьшится в два раза: \(\sigma_{new}=2, \sigma_{old}=1\) \(\rightarrow\) \(r_{new}=r_{old}/2\);
Это соответствует снижению концентрации одного из реагентов (действующих поверхностей - аналог действующих масс) в два раза.
Предэкспоненциальный множитель в \(c^{-1}\) может быть выражен следующим образом:

\[ A_s=A_{dim} \Gamma ^{(m-1)}/ \prod_i \sigma_i, \]

где i - реагент, m - число реагентов-поверхностных частиц.
Данным образом в mech_optimiz реализована работа с кинетическими параметрами для случая наличия частиц, занимающих несколько активных центров.
